# Figures for the JOSE Paper

This directory must contain two PNG figures before submitting to JOSE.
Both are generated by running the AEOP application with the parameters
described below. The filenames must match exactly what is referenced in
`paper.md`.

---

## Figure 1 — `fig1_altitude_plot.png`

**Caption in paper.md:**
> Altitude-versus-time plot generated by `plot_target_visibility()` for the
> Orion Nebula (M42) observed from Uberaba, Brazil (lat. −19.7°) on a
> representative winter night. The green shaded area marks the observation
> window above the 30° elevation threshold; the red dashed line marks the
> threshold itself.

**How to generate this figure:**

1. Run the Streamlit application:
   ```bash
   streamlit run app.py
   ```
2. In the sidebar:
   - **Location:** `Uberaba, Brazil`
   - **Date:** `2024-07-15` (mid-winter, M42 well-placed)
   - **Minimum elevation:** `30`
3. Under **Target Selection**, clear the preset list and type `M42`
4. Click **Generate Nightly Analysis**
5. Right-click the altitude plot → **Save image as**
   → save as `figures/fig1_altitude_plot.png`

**Alternative (Jupyter Notebook):**
```python
from src.location import get_location_from_city, set_timezone_for_sao_paulo
from src.targets import get_target_skycoords
from src.analysis import calculate_nightly_events, analyze_target_visibility_for_night
from src.plotting import plot_target_visibility
from datetime import date
import pytz
from astropy import units as u

location = get_location_from_city("Uberaba, Brazil")
tz = set_timezone_for_sao_paulo(location) or pytz.UTC
analysis_date = date(2024, 7, 15)
events = calculate_nightly_events(analysis_date, location, tz)
coords = get_target_skycoords(["M42"])
df = analyze_target_visibility_for_night(
    events['inicio_noite'], events['fim_noite'],
    location, coords['M42'], 30 * u.deg
)
fig = plot_target_visibility(df, "M42", analysis_date, 30)
fig.savefig("figures/fig1_altitude_plot.png", dpi=300, bbox_inches='tight')
```

**Required specifications:**
- Resolution: **300 DPI minimum**
- Format: **PNG**
- Approximate dimensions: 12 × 6 inches (default figure size)
- Ensure the x-axis time labels are legible at reduced size

## Figure 2 — `fig2_annual_heatmap.png`

**Caption in paper.md:**
> Annual visibility calendar heatmap generated by `plot_yearly_visibility()`
> for M42 (Orion Nebula) as seen from Uberaba, Brazil in 2024. Each cell
> encodes the number of hours the target was above 30° on that night; darker
> cells indicate longer visibility windows. The gap in June–July reflects the
> southern winter when Orion is a daytime object.

**How to generate this figure:**

1. Run the Streamlit application:
   ```bash
   streamlit run app.py
   ```
2. In the sidebar:
   - **Location:** `Uberaba, Brazil`
3. Select the **Annual Visibility Calendar** tab
4. **Target:** `M42`
5. **Year:** `2024`
6. Click **Generate Annual Calendar**
7. Right-click the heatmap → **Save image as**
   → save as `figures/fig2_annual_heatmap.png`

**Alternative (Jupyter Notebook):**
```python
from src.location import get_location_from_city, set_timezone_for_sao_paulo
from src.targets import get_target_skycoords
from src.analysis import analyze_year_visibility
from src.plotting import plot_yearly_visibility
import pytz
from astropy import units as u

location = get_location_from_city("Uberaba, Brazil")
tz = set_timezone_for_sao_paulo(location) or pytz.UTC
coords = get_target_skycoords(["M42"])
df_year = analyze_year_visibility(
    2024, location, tz, coords['M42'], 30 * u.deg
)
fig = plot_yearly_visibility(df_year, "M42", 2024)
fig.savefig("figures/fig2_annual_heatmap.png", dpi=300, bbox_inches='tight')
```

**Note:** The annual analysis iterates over 365 nights and may take
5–15 minutes depending on your hardware. This is expected behaviour.

**Required specifications:**
- Resolution: **300 DPI minimum**
- Format: **PNG**
- Approximate dimensions: 18 × 7 inches (default figure size)
- Month labels on the y-axis must be legible
